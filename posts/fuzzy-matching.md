## 为什么需要模糊匹配

供应商报的型号和我们系统里的 SKU 编码，经常存在细微差异：

- 多了空格：`A 123` vs `A123`
- 大小写不一致：`abc` vs `ABC`
- 多了连字符：`X-100` vs `X100`

精确匹配直接挂掉，模糊匹配就是来处理这些情况的。

## FuzzyWuzzy 常用方法

```python
from fuzzywuzzy import fuzz, process

# 简单相似度（顺序敏感）
fuzz.ratio("abc", "acb")  # 66

# 部分匹配（短字符串在长字符串里找）
fuzz.partial_ratio("abc", "xabcx")  # 100

# 排序后再比较（顺序不敏感）
fuzz.token_sort_ratio("新款 A100 红色", "A100 红色 新款")  # 100

# 从候选列表里找最佳匹配
process.extractOne("A123", ["A-123", "B456", "A124"])
# ('A-123', 91)
```

## 实际踩坑

### 坑 1：分数阈值设太低

设了 70 分就认为匹配，结果把完全不同的型号匹配在一起了。实践下来 **85 分**是个比较稳的阈值。

### 坑 2：候选列表太大导致很慢

`process.extractOne` 每次都要遍历整个候选列表。候选有 5000 条时，跑一批数据要好几分钟。

解决方案：先用规则过滤缩小候选范围（比如只取前几个字符相同的），再做模糊匹配。

### 坑 3：中文和英文混合

中文字符的 token 分割和英文不一样，`token_sort_ratio` 在中文场景下效果差。改用 `partial_ratio` 后好多了。
